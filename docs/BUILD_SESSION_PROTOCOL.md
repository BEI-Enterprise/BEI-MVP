## PRE-SESSION CHECKLIST

[ ] I have read the README.md
[ ] I know which phase I am building
[ ] I know what is in scope for this phase
[ ] I know what is NOT in scope for this phase
[ ] I am not skipping any phase
[ ] Every feature I am building serves Understand, Decide, Improve or Learn
[ ] I am not building UI before the intelligence beneath it exists
[ ] I am not deploying anything that bypasses the approval framework
[ ] Detection is not proof - I have not treated a detected constraint as verified
[ ] No pricing, staffing or strategic decisions will be auto-deployed

## DRIFT DETECTION

If at any point during a session the work being done:

- Does not serve Understand, Decide, Improve or Learn
- Skips ahead of the current phase
- Builds UI before intelligence exists beneath it
- Auto-deploys pricing, staffing or strategic decisions
- Treats a detected constraint as verified without passing the challenge framework
- Introduces a new industry outside the MVP 1 three
- Introduces a new constraint outside the MVP 1 ten

STOP. Return to the architecture. Realign before continuing.
